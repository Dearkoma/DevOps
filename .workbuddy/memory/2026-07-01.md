## 实例展开面板新增容器日志查看与保存

- **后端**：

    - K8s：`kubectl logs <pod> -n <ns> --tail=<tail>`，Pod 崩溃时自动回退 `--previous`

    - 复用已有的 `kubectl()` / `docker()` ProcessBuilder 安全执行方法
  - `InstanceController.java` 新增 `GET /api/instances/{id}/logs?tail=200`
- **前端**：
  - `api.js` 新增 `getInstanceLogs(id, tail)`
  - `InstanceList.jsx` 展开面板新增日志区域：
    - 查看日志按钮 → 调用 API 获取日志
    - 行数选择（100/200/500/1000行）
    - 刷新按钮、保存到文件按钮（下载 .txt）、关闭按钮
    - 日志以深色终端风格 `<pre>` 显示，maxHeight 400px 可滚动
  - 新增 state: `logsData`, `logsLoading`, `showLogsId`
  - 新增回调: `handleViewLogs`, `handleSaveLogs`, `handleCloseLogs`
  - 通过 `shared` 对象传递到所有三个视图组件（AllInstancesView/DockerView/K8sView）

## 修复 K8s 重启/停止/删除用 Pod 名当 Deployment 名
- **问题**：`kubectl rollout restart deployment/proj-xxx-7f55c7956f-qpwm2` 失败，因为 Deployment 名是 `proj-xxx`，不是 Pod 名
- **修复**：

  - `stopK8sInstance`：`kubectl scale deployment/<depName> --replicas=0 -n <ns>`
  - `deleteK8sResources`：`kubectl delete deployment <depName> -n <ns>`
  - 全部改用 `kubectl()` ProcessBuilder 传参数组

## 停止状态隐藏链接 + 日志显示停止前缓存
- **需求**：停止后不应显示链接（逻辑不通），日志应可查看之前的（给提示）
- **后端**：
  - `getAccessInfo`: STOPPED 状态直接返回 `{ stopped: true }`，不执行任何命令
  - `getContainerLogs`: K8s STOPPED 状态返回 `{ stopped: true }`，让前端用缓存
- **前端**：
  - 展开面板：STOPPED 显示黄色"实例已停止"提示条，不显示链接
  - 日志区域始终显示（独立于链接是否存在）
  - 新增 `logsCache` state：成功获取日志时按 instanceId 缓存
  - 停止后：有缓存 → 显示缓存日志 + 黄色警告条；无缓存 → 显示"无缓存日志"提示
  - 日志三种显示状态：正常(success) / 停止(stopped+缓存) / 错误(failed)
- **提交**: `5d238e1`

## 修复日志区域不渲染
- **问题**：用户反映"前端日志依旧没有"
- **根因**：日志区域渲染条件是 `{accessInfo && !accessLoading && (...)}`，当 `accessInfo` 为 `null` 时（API 调用失败或边缘情况），日志区域不渲染
- **修复**：改为 `{!accessLoading && (...)}`，只要加载完成就显示日志区域，独立于 `accessInfo`
- **额外**：`handleViewLogs` 添加 `console.log` 调试日志，用户可在浏览器 F12 Console 查看 API 返回
- **注意**：后端当前未运行（port 8080），用户需在 IDEA 重新启动后端
- **提交**: `1752811`
