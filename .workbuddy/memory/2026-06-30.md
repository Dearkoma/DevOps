## 定位并修复侧边栏消失根因（用户授权 AI 改代码+构建+git）
用户给出关键线索：F12 里 `<nav class="sidebar-nav"></nav>` 是空的（无子元素）。
- 根因定位：`AppLayout.jsx` 第 155 行 `userLevel = ROLE_LEVEL[user?.role] || 0`，
  当 localStorage 里 `devops_user` 为旧数据/脏数据（role 字段值不匹配 ADMIN/MANAGER/DEVELOPER/VIEWER）时，
  `ROLE_LEVEL[user.role]` 返回 `undefined`，`|| 0` 使 userLevel=0，
  所有菜单项（最低 VIEWER=1）被 `filter` 过滤光，导致 sidebar-nav 为空。
- 修复：改为 `userLevel = ROLE_LEVEL[user?.role] ?? (user ? ROLE_LEVEL.DEVELOPER : 0)`，
  即 user 已登录但 role 无效时降级到 DEVELOPER(2)，保证菜单可见。
- 提交 `dab02c0`：1 file changed, +2/-1
- Vite HMR 自动生效，用户刷新浏览器即可看到菜单
- 彻底解决需用户清 localStorage 重新登录（让后端重新返回正确的 role）

## 修复构建失败 - DB schema 与 entity 不一致（用户授权 AI 执行）

- `notifications: Field 'user_id' doesn't have a default value`

1. `service_instances.environment_id`: NOT NULL → NULL，CASCADE → SET NULL
2. `notifications.user_id`: NOT NULL → NULL（外键改 SET NULL）
3. 同步更新 `devops-platform/sql/init.sql`，重建库不再出此问题
