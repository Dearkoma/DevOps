## 2026-07-01 12:32 ~ 12:49 数据库名创建失败修复

### 问题根因
`DatabaseProvisioningService` 用 `DriverManager.getConnection()` 手动拼 JDBC URL 创建独立库。
`extractBaseJdbcUrl()` 去掉数据库名后生成的 URL `jdbc:mysql://localhost:3306?params...` 缺少端口后 `/`，
导致 MySQL JDBC 驱动连接失败 → "Could not create connection to database server" → 所有带自定义 dbName 的构建失败。

### 修复方案
**彻底抛弃手动 JDBC URL 解析**，直接复用 Spring Boot 已有的 HikariCP DataSource：
- 构造函数注入 `DataSource`，替代 `@Value` 读 URL/用户名/密码
- `databaseExists()`: `dataSource.getConnection()` 替代 `DriverManager.getConnection()`
- `createDatabase()`: 同上
- 删除 `extractBaseJdbcUrl()` 方法（不再需要 URL 解析）

### 验证
- `curl` POST 测试带 `dbName=test_custom_db_v3` → 成功返回 build 记录
- MySQL 确认数据库 `test_custom_db_v3` 已创建
- 提交: `d172b50`
